Concept definitions for FP-0086:

- `BenchmarkTask`: a read-only benchmark intent definition. It names task kind, allowed evidence contracts, required citations, freshness/limitation expectations, permitted next action posture, forbidden actions, privacy boundary, and proof expectation. It is not a runtime route, tool, eval dataset, model call, mission, or product answer.
- `BenchmarkCase`: a future synthetic-only case instance for one BenchmarkTask. It may exist only after SafeDemoDataPolicy and privacy checks are implemented. It must not contain real finance data, private-source-derived data, credentials, source-pack mutation, or generated advice.
- `EvidenceRecallTask`: a benchmark task that asks whether existing EvidenceIndex/V2C artifacts can be found and cited. It must require SourceAnchor or accepted derived refs and expose freshness, limitations, and permitted read-only next actions.
- `SourceCoverageTask`: a benchmark task that checks source coverage posture for supported, unsupported, missing, stale, failed, or not-indexed evidence. It must not reinterpret unsupported sources.
- `PolicyLookupTask`: a benchmark task for existing policy lookup posture over explicit policy sources. It must stay read-only and must not provide legal advice, policy advice, or broad corpus retrieval beyond supported contracts.
- `ReportTraceabilityTask`: a benchmark task that checks whether a stored mission/report/proof artifact traces to evidence cards, SourceAnchors, Finance Twin refs, CFO Wiki refs, proof bundles, freshness, and limitations. It must not create or release reports.
- `MonitorBoundaryTask`: a benchmark task that checks deterministic monitor boundary posture, source lineage, freshness/missing-source posture, limitations, severity rationale, and proof references. It must not rerun monitors, create alerts, create missions, or remediate anything.
- `UnsafeActionRefusalTask`: a benchmark task that verifies requests for writes, sends, uploads, approvals, provider calls, certification, payment, legal/audit/tax advice, source mutation, finance writes, generated advice, or autonomous action fail closed.
- `MissingCitationTask`: a benchmark task that verifies positive claims without SourceAnchor or accepted derived refs fail closed as missing-citation posture.
- `EvidenceFaithfulnessTask`: a benchmark task that verifies claims are supported only by selected citations and accepted refs, and that unsupported, stale, missing, conflicting, or raw full-file-dump-like evidence fails closed.
- `CommunityPackManifest`: a future manifest contract for describing a public/community pack without shipping any source files in the first implementation. It should carry schema version, owning Finance Plan, synthetic-only declaration, allowed task taxonomy, privacy review status, no-runtime boundary, no-source-pack-behavior boundary, and validation posture.
- `SafeDemoDataPolicy`: the first V2F primitive. It defines synthetic-only, non-private, not-real-company-derived, clearly labeled, reviewed, and no-real-finance-data requirements before any sample/demo/benchmark case can be checked in.
- `SyntheticFinanceSourcePolicy`: a stricter future policy for synthetic finance source material. It must require invented company/customer/vendor/payroll/tax/legal/board/lender facts and must not be derived from a real company by renaming fields or dates.
- `BenchmarkPrivacyBoundary`: the rule that benchmark artifacts must not include real company exports, private source text, credentials, tokens, database dumps, object-store files, screenshots with private data, or source-pack data derived from a real company.
- `ContributorChallenge`: a future public onboarding challenge that exercises read-only evidence reasoning without implying public launch, SaaS deployment, provider integration, certification, legal/audit/tax advice, finance writes, or autonomous action.
- `ArchitectureMap`: a benchmark-facing map from raw sources, Finance Twin, CFO Wiki, EvidenceIndex, V2C tools, V2D atlas, V2E bounded orchestration, proof bundles, freshness, limitations, and permitted next actions. It is documentation/proof posture, not a new runtime graph.
- `BenchmarkProof`: a future proof posture showing benchmark definitions remain separate from product runtime, use no real finance data, add no fixtures/datasets/source packs, make no OpenAI API/model calls, and expose no provider/certification/delivery/payment/customer-contact/legal/audit behavior.
- `BenchmarkNoRuntimeBoundary`: the invariant that benchmark/community artifacts cannot create product routes, UI, schema, migrations, package scripts, smoke aliases, mission behavior, monitor behavior, reports, approvals, provider jobs, delivery records, certification records, source mutations, finance writes, generated product prose, runtime-Codex finance output, or autonomous action.
