## Surprises & Discoveries

Baseline proof gates intentionally proved `fp0102Absent: true`. That was correct before this slice. FP-0102 must replace the hard FP-0102 absence rule with an exact docs-only endpoint/OAuth/remote-MCP architecture allowance while preserving FP-0103 absence.

Official OpenAI docs reinforce that a public/full MCP app path has separate concerns: MCP server endpoint and tool schema, ChatGPT UI resource registration, OAuth/auth metadata and per-tool security schemes, HTTPS deployment, Developer Mode connection/testing, submission, and security/privacy review. FP-0102 records architecture gates only; it does not implement any of those concerns.

OpenAI Developer Docs MCP tooling was not exposed as a callable read-only MCP tool in this thread, so official OpenAI web docs were used directly. No OpenAI Developers tool was used to create API keys. No OpenAI Platform key setup was used. No OpenAI API key was used.

The first FP-0102 bridge run correctly failed because the exact plan wording was missing several explicit negative-scope phrases and the proof helper's implementation-path scanner treated existing token/session file names as FP-0102 implementation. The correction was proof-gate-only: add the explicit plan phrases and narrow the implementation-path scan to endpoint/OAuth/remote-MCP/Apps-SDK/FP-0102 paths instead of generic token/session filenames.
