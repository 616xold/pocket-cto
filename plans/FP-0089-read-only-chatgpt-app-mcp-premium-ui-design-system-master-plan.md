## Decision Log

Decision: FP-0089 is safe to create only as docs-and-plan plus proof-gate compatibility.
Rationale: PR #248 is merged, FP-0088 is shipped, active docs support FP-0088 as docs-only premium UI/security readiness, required proof tools existed and passed before edits, no real finance data or public sample pack is required, and this plan can remain non-runtime.

Decision: FP-0089 does not authorize UI code.
Rationale: this slice defines future design-system artifacts, component taxonomy, accessibility bars, interaction rules, copy style, refusal-state design, evidence hierarchy, and design QA gates only. It requires a later UI implementation Finance Plan before any component code.

Decision: FP-0089 does not authorize Apps SDK iframe/UI code.
Rationale: Apps SDK iframe/UI work would create app-facing runtime and platform-specific implementation concerns. A later UI implementation Finance Plan must open that scope explicitly.

Decision: FP-0089 does not authorize public app implementation or public app submission.
Rationale: public app work requires hosted MCP/server posture, app management permissions, screenshots, test prompts, privacy/security review, review workflow, and public distribution intent. FP-0089 records no submission artifact, listing copy, screenshot, review prompt, public directory work, or app draft.

Decision: FP-0089 does not authorize remote MCP deployment, endpoint implementation, or OAuth implementation.
Rationale: those scopes add network exposure, authentication, authorization, token handling, operational security, audit logging, and public data disclosure risks. A later threat-model/security implementation plan must prove them before any endpoint exists.

Decision: FP-0089 preserves all authority lanes.
Rationale: the future UI must display raw-source, Finance Twin, CFO Wiki, EvidenceIndex, V2C, V2D, V2E, V2F, V2G, and FP-0088 posture without making the UI or model response a new source of truth.

Decision: future UI must be evidence-first, not advice-first.
Rationale: Pocket CFO's proof point is source-backed decisions with limitations. Future copy may be polished, but it must not generate financial advice, dark-pattern action prompts, or action-looking controls for forbidden actions.

Decision: proof gates must accept exactly one docs-only FP-0089 file.
Rationale: replacing `fp0089Absent` with `fp0089AbsentOrDocsOnlyBoundaryVerified`, a then-current FP-0090 absence guard, `premiumUiDesignSystemPlanBoundaryVerified`, `noUiImplementationFromFp0089`, `noAppsSdkIframeFromFp0089`, and `noEndpointOauthSubmissionFromFp0089` keeps the gate stronger than simple absence.

Decision: no replay event is added.
Rationale: this slice changes planning records, active docs, and validation gates only. It does not change product runtime state, mission state, source state, reports, approvals, or evidence artifacts.
