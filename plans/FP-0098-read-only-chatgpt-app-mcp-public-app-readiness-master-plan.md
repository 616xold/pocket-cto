## Surprises & Discoveries

Baseline V2F/V2G proof gates still proved `fp0098Absent: true` after shipped FP-0097. That is correct pre-edit behavior. This slice must replace hard FP-0098 absence with an exact docs-only public-app readiness boundary while preserving all local-only, read-only, no-runtime, no-real-finance-data, and no-public-app constraints.

Initial descriptor proof-gate smoke after creating FP-0098 caught missing exact machine-checkable wording for "not public ChatGPT App implementation", no source mutation, and no finance writes. The correction was confined to this plan text so the proof bridge could verify the intended docs-only boundary without widening scope.

Official OpenAI docs confirm that future public-app work is a separate platform implementation track. Apps in ChatGPT are built as MCP apps/servers with tools, resources, and optional UI resources; app submission has its own review, security, privacy, data-minimization, auth, and UX requirements; Developer Mode/full MCP connector testing can expose stronger tool-action and prompt-injection risks. That context supports a readiness master-plan only, not endpoint, OAuth, remote MCP, Apps SDK resource, or submission work in this slice.

Official web/plugin research used in this slice:

- OpenAI Apps SDK docs, `https://developers.openai.com/apps-sdk/`: used for current app/UI context and to confirm Apps SDK, MCP apps, resources, security/privacy, deployment, and app-submission are distinct platform concerns.
- OpenAI MCP Apps in ChatGPT docs, `https://developers.openai.com/apps-sdk/mcp-apps-in-chatgpt`: used to understand that apps are MCP servers with tool templates/resources and that UI/resource work needs a later implementation plan.
- OpenAI MCP server docs, `https://developers.openai.com/apps-sdk/concepts/mcp-server`: used for connector/tool/security-scheme context and to keep MCP server work future-only.
- OpenAI App Submission Guidelines, `https://developers.openai.com/apps-sdk/app-submission-guidelines`: used for submission-boundary planning only, including review, security/privacy, data minimization, auth/deauthorization, user control, no hidden behavior, and UX quality.
- OpenAI Developer Mode help, `https://help.openai.com/en/articles/12584461-developer-mode-apps-and-full-mcp-connectors-in-chatgpt-beta.svgz`: used for Developer Mode/full MCP connector context, including write-action risk and prompt-injection caution.
